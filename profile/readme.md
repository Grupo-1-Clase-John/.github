# ColGiTech

Bienvenidos a la organización ColGiTech. Este espacio centraliza los repositorios, prácticas y entregables del Grupo 1 de la clase dirigida por el instructor John Becerra, y sirve como punto de partida para colaborar, aprender y compartir recursos.

---

## Propósito
- Agrupar proyectos, ejercicios y recursos del curso.
- Facilitar la colaboración entre estudiantes y el profesor.
- Mantener plantillas y buenas prácticas comunes para todas las entregas.

---

## Repositorios (estructura recomendada)
- `project-<nombre>` — Proyectos principales y entregables.
- `exercise-<tema>` — Prácticas y ejercicios por tema.
- `resources` — Guías, plantillas y material de apoyo.
- `infrastructure` — Scripts y configuración común de CI/CD.
- `docs` — Documentación general y políticas del grupo.

---

## Cómo publicar este README en la página de la organización
Para que este README aparezca en la página principal de la organización en GitHub, crea un repositorio público con el mismo nombre de la organización (`Grupo-1-Clase-John`) y coloca este archivo `README.md` en la raíz.

---

## Primeros pasos
1. Clona el repositorio que te interese:
   - `git clone https://github.com/Grupo-1-Clase-John/<repo>.git`
2. Crea una rama para tu trabajo:
   - `git checkout -b feat/<descripción-corta>`
3. Haz commits claros y atómicos:
   - Ejemplo: `feat(api): añadir endpoint /login`

---

## Flujo de trabajo recomendado
- Usar Issues para planificar tareas y reportar bugs.
- Crear Pull Requests desde ramas con nombre descriptivo.
- Etiquetar PRs como `WIP`, `ready`, `review` según estado.
- Al menos 1 revisor antes de fusionar (o según indicación del profesor).
- Preferible: "Squash and merge" para PRs de cambios pequeños/atemporales.

---

## Estándares y calidad
- Añadir pruebas unitarias cuando corresponda.
- Mantener linters y formateadores configurados (ESLint/Prettier, flake8/black).
- Documentar APIs y comportamientos públicos mínimos.
- Integrar checks automáticos con GitHub Actions (build, test, lint).

---

## Plantillas y archivos recomendados
- ISSUE_TEMPLATE.md — reporte estructurado (pasos, resultado esperado, entorno).
- PULL_REQUEST_TEMPLATE.md — resumen, cómo probar, checklist.
- CODE_OF_CONDUCT.md — normas de convivencia.
- CODEOWNERS — asignación automática de revisores.
- LICENSE — sugerido MIT (o la licencia indicada por el curso).

---

## Contribuciones
- Abrir un Issue antes de empezar tareas grandes.
- PRs pequeñas y enfocadas para facilitar revisiones.
- Respetar plantillas y convenciones de commit/branch.
- Mantener comunicación clara en los comentarios del PR.

---

## Comunicación y coordinación
- Indicar canal principal (Slack/Discord/Teams) y calendario de entregas.
- Responsable / contacto: John (profesor) y los coordinadores del grupo (añadir emails/handles).

---

## Mantenimiento
- Revisar y actualizar dependencias periódicamente.
- Cerrar issues antiguos y actualizar documentación.
- Mantener un CHANGELOG sencillo para entregas importantes.
